# AI-PLAT用户手册

## 第二部分：核心功能操作指南

本册是 AI-PLAT 客户试用手册的第二部分，适合已经完成账号登录、项目进入和基础环境确认的用户阅读。通过本册，读者可以按照核心功能流程完成项目、组件、数据、标注、工作流、训练、评估、发布和部署操作。

阅读衔接：阅读本册前，建议已完成《AI-PLAT客户试用使用文档（一）：试用准备与平台入门》中的账号和项目准备；完成本册后，可继续阅读《AI-PLAT客户试用使用文档（三）：场景教程与支持附录》，用于培训演示、问题排查和试用收尾。

---

## 1. 核心功能快速上手

推荐试用主线：创建项目并添加成员 -> 从市集获取算法组件 -> 上传训练或测试资源 -> 创建标签标注任务 -> 创建模型训练工作流 -> 运行模型预测和模型评估 -> 发布模型 -> 创建快速 Demo 或 API 部署进行验证。

### 1.1 新建项目并添加成员

功能用途：项目用于承载后续资源管理、数据处理、模型生产和部署管理操作。

权限说明：新建项目通常需要超级管理员或具备项目创建权限的账号。如当前账号无权限，请联系运维人员或管理员。

操作入口：左侧按钮栏【项目】-> 右上角【新建】或【新建项目】。

操作步骤：

1. 点击左侧【项目】。
2. 在项目列表页点击右上角【新建】。
3. 填写项目名称和项目简介。
4. 选择项目所有者。
5. 添加项目成员。
6. 根据项目用途选择或分配算力服务器。
7. 设置服务器使用时长，或设置为不限时长。
8. 点击【完成】。

配置说明：

| 字段 | 说明 |
| --- | --- |
| 名称 | 项目的名称，必填 |
| 简介 | 项目简要描述，选填 |
| 所有者 | 项目最高权限负责人，必填 |
| 添加成员 | 可在创建时添加，也可创建后添加 |
| 算力服务器 | 用于训练、预测、评估或部署 |

添加成员入口：项目内【设置】->【成员管理】->【添加】。

添加成员步骤：进入项目 ->【设置】->【成员管理】->【添加】或【添加成员】-> 输入用户名或邮箱 -> 选择账号 ->【确认】-> 根据需要设置成员角色。

常见项目角色：

| 角色 | 说明 |
| --- | --- |
| 所有者 owner | 项目的拥有者，拥有项目最高权限，项目中唯一 |
| 管理员 master | 项目管理者，可添加成员、配置权限等 |
| 普通成员 member | 默认拥有一定权限，具体权限由项目管理员配置 |
| AI 生产工程师 | 通常负责数据、标注、训练、评估等生产流程 |
| AI 运维工程师 | 通常负责发布、部署、监控和运维 |

### 1.2 获取算法组件

功能用途：组件是完成工作流的基本算法单元，可将输入资源按照配置进行处理，并输出目标资源。创建标签任务、图像处理、模型训练、模型预测、模型评估等操作通常都需要先将对应组件获取至项目。

操作入口：主页面【广场】或【资产广场】-> 选择组件 ->【申请】/【获取】。

操作步骤：进入【广场】->【资产广场】-> 按类型筛选目标组件 -> 点击【申请】或【获取】-> 在弹窗中选择项目 ->【确认】-> 等待审批通过，或由管理员直接完成获取。

注意事项：如找不到目标项目，请检查该项目是否已包含该组件。组织层、团队层、项目层的权限不同；无上层权限时，请联系管理员拉取组件。

查看项目组件：进入项目 ->【设置】->【资产获取】，查看已添加至项目的组件。

#### 上传表格数据集

操作步骤：进入数据集新建页面 -> 选择【新建数据集】 -> 类型选择【表格】 -> 上传表格文件 -> 根据需要配置分隔符、编码、列标题、跳过行 -> 在预览页面确认表格内容 -> 点击【完成】。

### 1.4 上传模型资源

功能用途：用于将已有模型上传到项目中，或为已有模型注册新版本。

操作入口：项目内【模型】->【新建】。

操作步骤：进入模型新建页面 -> 选择【新建模型】->【下一步】-> 填写模型属性 -> 上传模型文件 ->【完成】。

字段说明：

| 字段 | 说明 |
| --- | --- |
| 名称 | 长度 2-36 位字符，仅支持 `-`、`_` 两种特殊字符，且不能以特殊字符开头或结尾 |
| 任务类型 | 模型对应的任务类型 |
| 来源 | 支持直接上传 |
| 版本 | 默认从 V0 开始 |
| 项目内公开 | 设置项目内其他成员是否可见 |
| 标签列表 | 设置模型支持的标签列表 |
| 推理框架 | 填写模型推理框架 |
| 模型架构 | 选择模型真实框架 |

注意事项：支持 `.pkl`、`.pth`、`.pb`、`.ckpt`、`.onnx`、`.trt`。模型仅支持单个文件上传；如选择多个模型文件，默认使用第一个文件。第一次上传模型请选择【新建模型】；对已有模型迭代时请选择【新建模型新版本】。

### 1.5 下载数据集或模型

下载入口：进入项目后点击左侧【数据】或【模型】，在列表中点击资源右侧下载图标；也可以进入数据集或模型详情页，点击【下载】。

### 1.6 创建标签标注任务

功能用途：用于对图片进行人工标注，为后续模型训练提供训练标签。

前期准备：已完成账号登录、已加入项目、已上传待标注图片、已获取标签标注相关组件。

数据要求：图片支持 `.jpg`、`.jpeg`、`.png`、`.bmp`。

操作步骤：进入项目 -> 上传待标注图片资源 -> 进入数据清洗或标签任务相关入口 -> 点击创建标签任务 -> 配置任务名称、任务类型、标签、负责人等信息 ->【创建】-> 标注负责人在待办列表中打开任务 -> 进入 Labeling 标注页面 -> 完成标注 ->【提交任务】-> 回到资源页面查看或下载标签资源。

### 1.7 使用 Labeling 标注工具

Labeling 是 AI-PLAT 的在线标签标注工具，主要用于图片标注和审核。

页面组成：

| 区域 | 说明 |
| --- | --- |
| 标题栏 | 查看当前任务名称 |
| 左侧工具栏 | 添加、复制、删除、显示/隐藏标签、放大/缩小等 |
| 图片展示区 | 进行缺陷框、多边形或分类标注 |
| 标签列表 | 展示任务配置的可选标签 |
| 文件列表 | 展示分配到当前用户的图片 |
| 完成任务按钮 | 提交并结束当前任务 |

常用操作：添加标签、复制标签、删除标签、显示/隐藏标签、放大/缩小、上一页/下一页、无标签跳过、存疑。

支持页面类型：目标检测页面、图像分类页面、图像多标签分类页面、图像分割页面。

### 1.8 创建图像处理工作流

功能用途：用于对原始图片进行切图、亮度调整、对比度调整等预处理，使图片满足模型训练条件。若图片已经标注，标签文件有时也需要对应处理。

前期准备：已加入项目、已上传待处理图片、如有标签需准备与图片一一对应的标签文件、已获取图像处理组件，例如【固定区域切图】。

数据要求：图片格式为 `.jpg`、`.png`、`.bmp`；目标检测和分类标签常用 `.xml`；图像分割标签支持 `.xml` 或 `.json`。

操作步骤：上传待处理图片和标签资源 -> 从市集获取图像处理组件至项目 -> 进入项目【工作流】->【新建】->【新建工作流】-> 填写工作流名称、概述、说明 -> 进入画布 -> 将输入资源、图像处理组件、输出资源拖拽到画布 -> 连接节点 -> 配置输入输出资源 -> 配置组件参数 ->【保存】->【运行】-> 选择运行设备 -> 查看日志 -> 查看或下载输出资源。

注意事项：概述长度不超过 20 个字符。如项目内无设备，请联系运维人员添加设备。

### 1.9 创建模型训练工作流

功能用途：用于基于训练图片、标签和预训练模型训练新的深度学习模型。

前期准备：已加入项目、已上传训练图片、已上传训练标签、已准备预训练模型、已获取模型训练组件，例如【训练目标检测模型】。

数据要求：训练图片支持 `.jpg`、`.jpeg`、`.png`、`.bmp`。训练标签需与图片对应。目标检测标签通常为包含类别名和位置的 XML 文件。标签类型建议设置为英文字符。

操作步骤：上传训练所需资源 -> 从市集获取模型训练组件至项目 -> 进入项目【工作流】->【新建】->【新建工作流】-> 填写工作流信息 -> 进入画布 -> 拖拽训练图片、训练标签、预训练模型、训练组件和输出资源 -> 连接节点 -> 配置输入资源和输出资源 -> 配置训练组件参数 -> 如需高级调参点击【更多配置】->【保存】->【运行】-> 选择运行设备 -> 查看训练日志 -> 查看并下载输出资源。

模型训练常见输出资源：模型、模型配置文件、类别字典、训练报告、训练过程日志。

日志查看方式：工作流日志可在工作流右上角【日志】查看所有元件日志；元件日志可点击具体元件后进入【日志】查看。

### 1.10 创建模型预测工作流

功能用途：用于对训练好的模型进行批量预测，帮助客户观察模型在测试数据上的实际效果。

注意事项：AI-PLAT 暂不支持对本地上传的模型进行预测操作，仅支持对 AI-PLAT 生产出的模型进行预测操作。

前期准备：已上传测试图片、已有 AI-PLAT 生产出的模型、已获取模型预测组件，例如【目标检测模型预测】。

操作步骤：上传测试图片资源 -> 确认已有可用于预测的模型资源 -> 从市集获取模型预测组件 -> 新建工作流 -> 在画布中连接测试图片、模型、预测组件和输出资源 -> 配置输入输出资源 -> 配置预测组件参数 ->【保存】->【运行】-> 选择运行设备 -> 查看日志 -> 查看或下载预测输出资源。

常见输出资源：预测结果图片、预测标签、预测日志。

### 1.11 创建模型评估工作流

功能用途：用于评估模型预测效果，统计准确率、过检、漏检、混淆矩阵等指标，帮助判断模型是否达到生产要求。

注意事项：AI-PLAT 暂不支持对本地上传的模型进行评估操作，仅支持对 AI-PLAT 生产出的模型进行评估操作。

前期准备：已上传待评估图片、已准备答案标签、已准备待评估标签或待评估模型、已获取模型评估组件，例如【评估目标检测模型预测效果】。

操作步骤：上传评估所需资源 -> 从市集获取模型评估组件 -> 新建工作流 -> 在画布中连接评估图片、答案标签、待评估标签或模型、评估组件和输出资源 -> 配置输入输出资源 -> 配置评估组件参数 ->【运行】-> 选择运行设备 -> 查看日志 -> 查看评估报告。

常见输出资源：检测框图片、评估报告文本、可视化评估报告。

评估报告常见内容：

| 报告类型 | 主要内容 |
| --- | --- |
| 目标检测评估报告 | 报告基本信息、整图评估结果、缺陷详细统计、过检漏检、缺陷分布热力图 |
| 图像分类评估报告 | 报告基本信息、评估指标、Recall/Precision 混淆矩阵、真实-预测柱状图、误检图片 |

查看可视化报告：进入工作流点击左侧资源名称跳转资源详情，或进入资源模块点击资源名称进入详情，在详情页点击【预览】查看并下载报告。

### 1.12 运行配置与设备选择

| 运行方式 | 说明 |
| --- | --- |
| 立即运行 | 工作流配置完成后立即运行，较短时间内输出结果 |
| 预约运行 | 预约 7 天内的可用时间段运行，便于利用空余算力 |

立即运行：工作流绘制完成并配置参数后，点击【运行】->【立即运行】-> 选择运行设备 -> 开始运行。

预约运行：工作流绘制完成并配置参数后，点击【运行】->【预约运行】-> 选择 7 天内日期和开始时间 -> 点击【预约】。预约成功后，页面会显示任务开始时间和预计运行时间。

设备选择说明：全局配置为整个工作流选择运行设备；局部配置为工作流内单个组件选择运行设备。如项目内无可用设备，请联系运维人员添加或分配设备。

### 1.13 发布模型

功能用途：用于将精度达标的模型发布，发布后可进行上下架管理，并用于后续快速部署或生产部署。

操作入口：项目内【模型】-> 模型详情 -> 发布模块【发布】。

操作步骤：进入项目【模型】页面 -> 选择需要发布的模型并进入模型详情页 -> 在发布模块点击【发布】-> 填写发布相关属性信息 ->【完成】提交发布申请 -> 管理员在发布管理模块审批 -> 审批通过后模型完成发布。

注意事项：普通成员发起发布后，需管理员审批；管理员发起发布通常自动审批。处于上架状态的模型支持后续推送或部署；处于下架状态的模型不支持后续推送。

### 1.14 快速部署模型

功能用途：快速部署用于对已训练完成并发布上架的模型进行快速验证。平台支持快速 Demo 和 API 部署。

使用场景：通过快速 Demo 验证模型效果，或通过 API 部署调试和调用模型服务。

注意事项：Web 页面形式的快速 Demo 目前仅支持目标检测模型；API 调试支持目标检测、分类、分割三种应用场景。

#### 新建快速 Demo

操作入口：项目内【快速部署】->【新建】->【快速Demo】。

操作步骤：进入项目【快速部署】->【新建】->【快速Demo】-> 选择模型发布名称 -> 填写部署名称 -> 选择限制时长和可用于快速部署的设备 -> 配置模型类别和置信度阈值 ->【完成】-> 启动成功后复制 URL -> 打开 URL 通过 Web 页面测试模型。

快速 Demo 页面通常包含图片列表、主图片展示区域、识别结果、原始结果。单次仅允许上传 10 张图片。目标检测/分割场景可调整置信度阈值；分类场景置信度阈值不可更改。

#### 新建 API 部署

操作入口：项目内【快速部署】->【新建】->【API部署】。

操作步骤：进入项目【快速部署】->【新建】->【API部署】-> 选择模型发布名称 -> 填写部署名称，平台自动生成 Token -> 选择限制时长和设备 -> 配置模型类别和置信度阈值 ->【完成】-> 启动成功后复制 Token，用于本地调试。

API 部署详情页通常包含部署信息、Token、启动时间、终止时间、剩余时长、接口用例、参数信息、运行历史和模型信息。

### 1.15 生产部署

功能用途：生产部署用于将成熟产品或部署方案部署至生产环境，并支持更新、回退、启动、终止、删除、日志监控和设备监控。

使用场景：现场通过 AI-PLAT 部署成熟产品，例如 PVVI、SID 等；用户需要可视化查看生产环境设备和服务日志；更新后的服务异常时，需要快速回退到历史版本。

#### 获取产品至项目

操作入口：市集【公共产品库】-> 产品 ->【获取】。

操作步骤：进入市集 -> 打开【公共产品库】-> 找到目标产品 -> 点击【获取】-> 在弹窗中选择目标项目 -> 点击【确认】。

注意事项：AI-PLAT 项目仅支持关联单个产品，产品关联后通常不支持修改。

#### 新建生产部署

操作入口：项目内【生产部署】-> 部署仓库 ->【新建部署】，或项目内【生产部署】->【新建】。

操作步骤：进入项目【生产部署】-> 选择部署方案并新建部署 -> 填写部署名称 -> 选择部署方案和服务 -> 配置服务信息 -> 选择模型 -> 选择智能部署或指定部署 -> 配置设备或实例数 -> 填写参数配置 ->【完成】。部署新建完成后会自动启动。

部署方式说明：

| 部署方式 | 说明 |
| --- | --- |
| 智能部署 | 无需指定到具体设备显卡，只需填写部署实例数 |
| 指定部署 | 需要选择具体设备和显卡，并配置实例数 |

#### 生产部署管理

| 操作 | 说明 |
| --- | --- |
| 更新 | 修改模型、实例数、设备、参数等 |
| 回退 | 更新后异常或精度偏差较大时，恢复至历史版本 |
| 启动 | 对已终止或异常状态部署进行启动 |
| 终止 | 停止运行中的部署 |
| 删除 | 删除终止或异常状态的部署 |
| 操作记录 | 查看启动、终止、更新、回退等操作历史 |
| 日志监控 | 以服务或实例维度查看日志并下载 |
| 设备监控 | 查看实例运行状态和生产设备状态 |
