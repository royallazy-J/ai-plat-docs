AI-PLAT用户手册

第一部分：试用准备与平台入门

本册是 AI-PLAT 客户试用手册的第一部分，适合首次接触平台的客户阅读。通过本册，读者可以了解平台定位、适用对象、试用前准备、账号登录配置以及项目入口。

阅读衔接：本册完成后，建议继续阅读《AI-PLAT客户试用使用文档（二）：核心功能操作指南》，进入数据、组件、工作流、模型和部署等实际操作。

1. 文档说明

1.1 文档目标

本文档用于指导客户快速熟悉 AI-PLAT 平台，并通过一条完整的试用流程，完成从账号注册、登录、项目创建、资源上传、组件获取、工作流运行到模型发布和部署验证的基础操作。

客户可以按照本文档顺序完成一次完整的平台试用，也可以根据自身角色直接进入对应章节操作。

1.2 平台简介

AI-PLAT 是面向企业 AI 开发和维护的低代码平台，帮助企业自主开发和维护 AI，实现设备故障预测分析、工艺参数优化、缺陷检测识别等 AI 场景。

平台可以用于两类典型场景：

  • AI 模型维护：当企业已有 AI 模型投入使用后，若场景变化或模型精度偏移，可通过 AI-PLAT 自主维护模型，使模型重新适配业务现场。
  • AI 应用开发：当企业需要建设新的 AI 应用时，可通过 AI-PLAT 完成数据准备、标注、训练、评估、部署等完整流程。

1.3 核心功能模块

2. 适用对象

2.1 企业管理员

企业管理员主要负责项目创建、成员管理、权限配置、算力设备分配、资源审批、发布审批等管理工作。

2.2 AI 生产工程师

AI 生产工程师主要负责数据上传、标签标注、组件拉取、工作流创建、模型训练、模型预测和模型评估。

2.3 AI 运维工程师

AI 运维工程师主要负责模型发布、快速部署、API 部署、生产部署、部署监控、日志排查和服务更新回退。

2.4 普通业务用户

普通业务用户主要参与试用验证、数据准备、标注任务执行、模型 Demo 测试和试用反馈。

4. 平台登录与账号配置

4.1 注册 AI-PLAT 账号

功能用途：用于客户首次获得 AI-PLAT 登录账号。

操作步骤：

  1. 使用 Edge 或 64 位 Chrome 浏览器访问平台地址。
  2. 进入登录页面后，点击右上角【点击注册】。
  3. 在注册页面输入真实邮箱地址、用户名、手机号、密码，并二次确认密码。
  4. 点击【立即注册】。
  5. 打开注册邮箱，查收来自 AI-PLAT 的验证邮件。
  6. 点击邮件中的验证链接，完成注册。

注意事项：

  • 密码长度为 8-18 位字符，且必须包含字母和数字。
  • 如未收到验证邮件，请先检查垃圾箱。
  • 如仍未收到，请等待 60 秒后点击【重新发送】。

预期结果：注册成功后，用户可以使用注册邮箱登录 AI-PLAT。

4.2 登录 AI-PLAT

操作步骤：

  1. 打开 AI-PLAT 登录页面。
  2. 输入注册邮箱或用户名。
  3. 输入密码。
  4. 点击【登录】。

登录页面示例：

注意事项：首次使用 AI-PLAT 的新用户，建议使用注册邮箱登录。修改用户名后，也可以使用用户名登录。登录页面支持隐藏/显示密码。勾选【自动登录】后，10 天内可默认登录。

4.3 修改用户名

操作步骤：登录 AI-PLAT 后点击右上角个人图标 ->【设置】->【基本信息】-> 点击用户名输入框或用户名修改按钮 -> 修改用户名后点击【确定】或保存。

注意事项：用户名长度为 2-36 字节。修改用户名后，可以使用用户名和密码登录。

4.4 找回密码

操作步骤：

  1. 在 AI-PLAT 登录页面右下角点击【忘记密码】。
  2. 输入已注册的邮箱账号。
  3. 点击【确认】。
  4. 打开注册邮箱，查收找回密码邮件。
  5. 点击邮件中的验证链接，进入重置密码页面。
  6. 输入新密码并再次确认。
  7. 点击【确认】，完成密码重置。

注意事项：密码长度为 8-18 位字符，且必须包含字母和数字。如未收到邮件，请检查垃圾箱或等待 60 秒后重新发送。

1. 核心功能快速上手

推荐试用主线：创建项目并添加成员 -> 从市集获取算法组件 -> 上传训练或测试资源 -> 创建标签标注任务 -> 创建模型训练工作流 -> 运行模型预测和模型评估 -> 发布模型 -> 创建快速 Demo 或 API 部署进行验证。

1.1 新建项目并添加成员

功能用途：项目用于承载后续资源管理、数据处理、模型生产和部署管理操作。

权限说明：新建项目通常需要超级管理员或具备项目创建权限的账号。如当前账号无权限，请联系运维人员或管理员。

操作入口：左侧按钮栏【项目】-> 右上角【新建】或【新建项目】。

操作步骤：

  1. 点击左侧【项目】。
  2. 在项目列表页点击右上角【新建】。
  3. 填写项目名称和项目简介。
  4. 选择项目所有者。
  5. 添加项目成员。
  6. 根据项目用途选择或分配算力服务器。
  7. 设置服务器使用时长，或设置为不限时长。
  8. 点击【完成】。

配置说明：

字段说明
名称项目的名称，必填
简介项目简要描述，选填
所有者项目最高权限负责人，必填
添加成员可在创建时添加，也可创建后添加
算力服务器用于训练、预测、评估或部署

添加成员入口：项目内【设置】->【成员管理】->【添加】。

添加成员步骤：进入项目 ->【设置】->【成员管理】->【添加】或【添加成员】-> 输入用户名或邮箱 -> 选择账号 ->【确认】-> 根据需要设置成员角色。

常见项目角色：

角色说明
所有者 owner项目的拥有者，拥有项目最高权限，项目中唯一
管理员 master项目管理者，可添加成员、配置权限等
普通成员 member默认拥有一定权限，具体权限由项目管理员配置
AI 生产工程师通常负责数据、标注、训练、评估等生产流程
AI 运维工程师通常负责发布、部署、监控和运维

1.2 获取算法组件

功能用途：组件是完成工作流的基本算法单元，可将输入资源按照配置进行处理，并输出目标资源。创建标签任务、图像处理、模型训练、模型预测、模型评估等操作通常都需要先将对应组件获取至项目。

操作入口：主页面【广场】或【资产广场】-> 选择组件 ->【申请】/【获取】。

操作步骤：进入【广场】->【资产广场】-> 按类型筛选目标组件 -> 点击【申请】或【获取】-> 在弹窗中选择项目 ->【确认】-> 等待审批通过，或由管理员直接完成获取。

注意事项：如找不到目标项目，请检查该项目是否已包含该组件。组织层、团队层、项目层的权限不同；无上层权限时，请联系管理员拉取组件。

查看项目组件：进入项目 ->【设置】->【资产获取】，查看已添加至项目的组件。

1.6 创建标签标注任务

功能用途：用于对图片进行人工标注，为后续模型训练提供训练标签。

数据要求：图片支持 .jpg、.jpeg、.png、.bmp。

操作步骤：进入项目 -> 上传待标注图片资源 -> 进入数据清洗或标签任务相关入口 -> 点击创建标签任务 -> 配置任务名称、任务类型、标签、负责人等信息 ->【创建】-> 标注负责人在待办列表中打开任务 -> 进入 Labeling 标注页面 -> 完成标注 ->【提交任务】-> 回到资源页面查看或下载标签资源。

1.8 创建图像处理工作流

功能用途：用于对原始图片进行切图、亮度调整、对比度调整等预处理，使图片满足模型训练条件。若图片已经标注，标签文件有时也需要对应处理。

前期准备：已加入项目、已上传待处理图片、如有标签需准备与图片一一对应的标签文件、已获取图像处理组件，例如【固定区域切图】。

数据要求：图片格式为 .jpg、.png、.bmp；目标检测和分类标签常用 .xml；图像分割标签支持 .xml 或 .json。

操作步骤：上传待处理图片和标签资源 -> 从市集获取图像处理组件至项目 -> 进入项目【工作流】->【新建】->【新建工作流】-> 填写工作流名称、概述、说明 -> 进入画布 -> 将输入资源、图像处理组件、输出资源拖拽到画布 -> 连接节点 -> 配置输入输出资源 -> 配置组件参数 ->【保存】->【运行】-> 选择运行设备 -> 查看日志 -> 查看或下载输出资源。

注意事项：概述长度不超过 20 个字符。如项目内无设备，请联系运维人员添加设备。

1.9 创建模型训练工作流

功能用途：用于基于训练图片、标签和预训练模型训练新的深度学习模型。

前期准备：已加入项目、已上传训练图片、已上传训练标签、已准备预训练模型、已获取模型训练组件，例如【训练目标检测模型】。

数据要求：训练图片支持 .jpg、.jpeg、.png、.bmp。训练标签需与图片对应。目标检测标签通常为包含类别名和位置的 XML 文件。标签类型建议设置为英文字符。

操作步骤：上传训练所需资源 -> 从市集获取模型训练组件至项目 -> 进入项目【工作流】->【新建】->【新建工作流】-> 填写工作流信息 -> 进入画布 -> 拖拽训练图片、训练标签、预训练模型、训练组件和输出资源 -> 连接节点 -> 配置输入资源和输出资源 -> 配置训练组件参数 -> 如需高级调参点击【更多配置】->【保存】->【运行】-> 选择运行设备 -> 查看训练日志 -> 查看并下载输出资源。

模型训练常见输出资源：模型、模型配置文件、类别字典、训练报告、训练过程日志。

日志查看方式：工作流日志可在工作流右上角【日志】查看所有元件日志；元件日志可点击具体元件后进入【日志】查看。

1.10 创建模型预测工作流

功能用途：用于对训练好的模型进行批量预测，帮助客户观察模型在测试数据上的实际效果。

注意事项：AI-PLAT 暂不支持对本地上传的模型进行预测操作，仅支持对 AI-PLAT 生产出的模型进行预测操作。

前期准备：已上传测试图片、已有 AI-PLAT 生产出的模型、已获取模型预测组件，例如【目标检测模型预测】。

操作步骤：上传测试图片资源 -> 确认已有可用于预测的模型资源 -> 从市集获取模型预测组件 -> 新建工作流 -> 在画布中连接测试图片、模型、预测组件和输出资源 -> 配置输入输出资源 -> 配置预测组件参数 ->【保存】->【运行】-> 选择运行设备 -> 查看日志 -> 查看或下载预测输出资源。

常见输出资源：预测结果图片、预测标签、预测日志。

1.11 创建模型评估工作流

功能用途：用于评估模型预测效果，统计准确率、过检、漏检、混淆矩阵等指标，帮助判断模型是否达到生产要求。

注意事项：AI-PLAT 暂不支持对本地上传的模型进行评估操作，仅支持对 AI-PLAT 生产出的模型进行评估操作。

前期准备：已上传待评估图片、已准备答案标签、已准备待评估标签或待评估模型、已获取模型评估组件，例如【评估目标检测模型预测效果】。

操作步骤：上传评估所需资源 -> 从市集获取模型评估组件 -> 新建工作流 -> 在画布中连接评估图片、答案标签、待评估标签或模型、评估组件和输出资源 -> 配置输入输出资源 -> 配置评估组件参数 ->【运行】-> 选择运行设备 -> 查看日志 -> 查看评估报告。

常见输出资源：检测框图片、评估报告文本、可视化评估报告。

评估报告常见内容：

报告类型主要内容
目标检测评估报告报告基本信息、整图评估结果、缺陷详细统计、过检漏检、缺陷分布热力图
图像分类评估报告报告基本信息、评估指标、Recall/Precision 混淆矩阵、真实-预测柱状图、误检图片

查看可视化报告：进入工作流点击左侧资源名称跳转资源详情，或进入资源模块点击资源名称进入详情，在详情页点击【预览】查看并下载报告。

1.12 运行配置与设备选择

运行方式说明
立即运行工作流配置完成后立即运行，较短时间内输出结果
预约运行预约 7 天内的可用时间段运行，便于利用空余算力

立即运行：工作流绘制完成并配置参数后，点击【运行】->【立即运行】-> 选择运行设备 -> 开始运行。

预约运行：工作流绘制完成并配置参数后，点击【运行】->【预约运行】-> 选择 7 天内日期和开始时间 -> 点击【预约】。预约成功后，页面会显示任务开始时间和预计运行时间。

设备选择说明：全局配置为整个工作流选择运行设备；局部配置为工作流内单个组件选择运行设备。如项目内无可用设备，请联系运维人员添加或分配设备。

1.13 发布模型

功能用途：用于将精度达标的模型发布，发布后可进行上下架管理，并用于后续快速部署或生产部署。

操作入口：项目内【模型】-> 模型详情 -> 发布模块【发布】。

操作步骤：进入项目【模型】页面 -> 选择需要发布的模型并进入模型详情页 -> 在发布模块点击【发布】-> 填写发布相关属性信息 ->【完成】提交发布申请 -> 管理员在发布管理模块审批 -> 审批通过后模型完成发布。

注意事项：普通成员发起发布后，需管理员审批；管理员发起发布通常自动审批。处于上架状态的模型支持后续推送或部署；处于下架状态的模型不支持后续推送。

1.14 快速部署模型

功能用途：快速部署用于对已训练完成并发布上架的模型进行快速验证。平台支持快速 Demo 和 API 部署。

使用场景：通过快速 Demo 验证模型效果，或通过 API 部署调试和调用模型服务。

注意事项：Web 页面形式的快速 Demo 目前仅支持目标检测模型；API 调试支持目标检测、分类、分割三种应用场景。

新建快速 Demo

操作入口：项目内【快速部署】->【新建】->【快速Demo】。

操作步骤：进入项目【快速部署】->【新建】->【快速Demo】-> 选择模型发布名称 -> 填写部署名称 -> 选择限制时长和可用于快速部署的设备 -> 配置模型类别和置信度阈值 ->【完成】-> 启动成功后复制 URL -> 打开 URL 通过 Web 页面测试模型。

快速 Demo 页面通常包含图片列表、主图片展示区域、识别结果、原始结果。单次仅允许上传 10 张图片。目标检测/分割场景可调整置信度阈值；分类场景置信度阈值不可更改。

新建 API 部署

操作入口：项目内【快速部署】->【新建】->【API部署】。

操作步骤：进入项目【快速部署】->【新建】->【API部署】-> 选择模型发布名称 -> 填写部署名称，平台自动生成 Token -> 选择限制时长和设备 -> 配置模型类别和置信度阈值 ->【完成】-> 启动成功后复制 Token，用于本地调试。

API 部署详情页通常包含部署信息、Token、启动时间、终止时间、剩余时长、接口用例、参数信息、运行历史和模型信息。

1.15 生产部署

功能用途：生产部署用于将成熟产品或部署方案部署至生产环境，并支持更新、回退、启动、终止、删除、日志监控和设备监控。

使用场景：现场通过 AI-PLAT 部署成熟产品，例如 PVVI、SID 等；用户需要可视化查看生产环境设备和服务日志；更新后的服务异常时，需要快速回退到历史版本。

获取产品至项目

操作入口：市集【公共产品库】-> 产品 ->【获取】。

操作步骤：进入市集 -> 打开【公共产品库】-> 找到目标产品 -> 点击【获取】-> 在弹窗中选择目标项目 -> 点击【确认】。

注意事项：AI-PLAT 项目仅支持关联单个产品，产品关联后通常不支持修改。

新建生产部署

操作入口：项目内【生产部署】-> 部署仓库 ->【新建部署】，或项目内【生产部署】->【新建】。

操作步骤：进入项目【生产部署】-> 选择部署方案并新建部署 -> 填写部署名称 -> 选择部署方案和服务 -> 配置服务信息 -> 选择模型 -> 选择智能部署或指定部署 -> 配置设备或实例数 -> 填写参数配置 ->【完成】。部署新建完成后会自动启动。

部署方式说明：

部署方式说明
智能部署无需指定到具体设备显卡，只需填写部署实例数
指定部署需要选择具体设备和显卡，并配置实例数

1. 典型场景操作教程

1.1 场景一：完成一次缺陷检测模型训练试用

适用客户：质检团队、AI 生产团队、工业视觉场景客户。

操作流程：注册并登录 AI-PLAT -> 加入或创建项目 -> 上传缺陷图片数据集 -> 创建标签标注任务 -> 使用 Labeling 完成目标检测标注 -> 从市集获取目标检测训练组件 -> 创建模型训练工作流 -> 配置训练图片、标签、预训练模型和输出资源 -> 选择设备并运行训练 -> 查看训练日志和训练报告 -> 下载训练产出的模型资源。

1.2 场景二：完成模型预测和评估

操作流程：上传测试图片 -> 选择 AI-PLAT 训练产出的模型 -> 获取模型预测组件 -> 创建模型预测工作流 -> 运行预测并查看预测输出 -> 准备答案标签和预测标签 -> 获取模型评估组件 -> 创建模型评估工作流 -> 运行评估 -> 查看可视化评估报告 -> 判断模型是否达到试用预期。

1.3 场景三：快速部署模型进行 Demo 验证

操作流程：选择已发布并上架的模型 -> 进入【快速部署】-> 新建快速 Demo -> 选择模型发布名称 -> 配置部署名称、限制时长和设备 -> 配置类别和置信度阈值 -> 完成部署 -> 复制 Demo URL -> 打开 Web 页面上传测试图片 -> 查看模型识别结果和原始结果。

1.4 场景四：生产环境部署和运维

操作流程：从公共产品库获取产品至项目 -> 在项目内关联产品 -> 进入生产部署模块 -> 选择部署方案并新建部署 -> 填写部署信息和服务配置 -> 选择智能部署或指定部署 -> 完成参数配置并启动部署 -> 查看部署状态 -> 通过日志监控和设备监控观察运行情况 -> 如有异常，执行更新、回退、终止或启动操作。

2. 常见问题 FAQ

2.1 登录失败怎么办？

可能原因：邮箱、用户名或密码输入错误；账号未完成邮箱验证；账号未开通或被停用；网络无法访问平台。

处理方式：检查账号和密码；首次登录优先使用注册邮箱；检查邮箱是否完成验证；点击【忘记密码】重置密码；联系管理员或客户成功经理确认账号状态。

2.2 注册后收不到验证邮件怎么办？

检查垃圾箱，确认邮箱地址是否填写正确，等待 60 秒后点击【重新发送】。仍未收到时联系平台支持人员。

2.3 找不到项目怎么办？

可能原因：尚未加入项目、项目被删除、当前账号权限不足、当前处于错误的组织或团队层级。

处理方式：联系项目管理员添加成员，检查当前所在组织或团队，请管理员确认项目是否存在并确认账号角色和权限。

2.4 无法新建项目怎么办？

当前账号可能不是超级管理员，或没有项目创建权限。请联系超级管理员创建项目，或联系运维人员开通权限。

2.5 找不到需要的组件怎么办？

可能原因：当前层级无组件权限，组件尚未从组织层或团队层拉取，项目已存在该组件但未在当前入口显示。

处理方式：在资产广场按组件类型筛选；进入项目【设置】->【资产获取】查看已获取组件；联系管理员从组织层或团队层拉取组件。

2.6 上传数据集失败怎么办？

可能原因：文件格式不支持、文件大小超过限制、文件内容为空或损坏、文件命名不符合要求、网络连接异常。

处理方式：检查文件格式，检查数据集名称是否符合 2-36 位字符规则，拆分或压缩过大的文件，重新导出文件后上传，联系技术支持排查。

2.7 标签标注任务打不开怎么办？

可能原因：当前账号不是任务负责人、任务已终止、图片资源或标签配置异常、浏览器兼容性问题。

处理方式：检查待办列表是否有该任务，联系任务创建人确认负责人配置，检查任务状态，使用 Edge 或 64 位 Chrome 重新打开。

2.8 工作流无法运行怎么办？

可能原因：输入输出资源未配置完整、组件参数未保存、项目内无可用设备、当前账号没有运行权限。

处理方式：检查工作流节点连接是否完整，检查所有输入输出资源是否已配置，点击组件确认参数配置并保存，确认项目内有可用设备，联系管理员检查权限。

2.9 模型预测或评估无法使用本地上传模型怎么办？

AI-PLAT 暂不支持对本地上传模型进行预测或评估操作，仅支持对 AI-PLAT 生产出的模型进行预测和评估。请先使用 AI-PLAT 完成模型训练，再使用训练产出的模型创建预测或评估工作流。

2.10 快速 Demo 无法测试怎么办？

可能原因：模型未发布、模型未上架、快速部署服务未启动成功、快速 Demo 类型不支持当前模型、单次上传图片数量超过限制。

处理方式：确认模型已发布并处于上架状态，检查快速部署详情页运行状态，目标检测模型优先使用快速 Demo，单次上传图片不超过 10 张，查看运行日志或联系运维人员。
