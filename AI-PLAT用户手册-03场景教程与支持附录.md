## 1. 典型场景操作教程

### 1.1 场景一：完成一次缺陷检测模型训练验证

适用客户：质检团队、AI 生产团队、工业视觉场景客户。

操作流程：注册并登录 AI-PLAT -> 加入或创建项目 -> 上传缺陷图片数据集 -> 创建标签标注任务 -> 使用 Labeling 完成目标检测标注 -> 从市集获取目标检测训练组件 -> 创建模型训练工作流 -> 配置训练图片、标签、预训练模型和输出资源 -> 选择设备并运行训练 -> 查看训练日志和训练报告 -> 下载训练产出的模型资源。

### 1.2 场景二：完成模型预测与效果检查

操作流程：上传测试图片 -> 选择 AI-PLAT 训练产出的模型 -> 获取模型预测组件 -> 创建模型预测工作流 -> 配置测试图片、模型和预测输出资源 -> 运行预测并查看预测输出 -> 在预测结果数据集中检查画框图、标签结果和 HTML 报告 -> 如测试集已包含标签，结合报告中的指标判断模型是否达到业务验证预期。

### 1.3 场景三：快速部署模型进行 Demo 验证

操作流程：选择已发布并上架的模型 -> 进入【快速部署】-> 新建快速 Demo -> 选择模型发布名称 -> 配置部署名称、限制时长和设备 -> 配置类别和置信度阈值 -> 完成部署 -> 复制 Demo URL -> 打开 Web 页面上传测试图片 -> 查看模型识别结果和原始结果。

### 1.4 场景四：生产环境部署和运维

操作流程：从公共产品库获取产品至项目 -> 在项目内关联产品 -> 进入生产部署模块 -> 选择部署方案并新建部署 -> 填写部署信息和服务配置 -> 选择智能部署或指定部署 -> 完成参数配置并启动部署 -> 查看部署状态 -> 通过日志监控和设备监控观察运行情况 -> 如有异常，执行更新、回退、终止或启动操作。

## 2. 常见问题 FAQ

### 2.1 登录失败怎么办？

可能原因：邮箱、用户名或密码输入错误；账号未完成邮箱验证；账号未开通或被停用；网络无法访问平台。

处理方式：检查账号和密码；首次登录优先使用注册邮箱；检查邮箱是否完成验证；点击【忘记密码】重置密码；联系管理员或客户成功经理确认账号状态。

### 2.2 注册后收不到验证邮件怎么办？

检查垃圾箱，确认邮箱地址是否填写正确，等待 60 秒后点击【重新发送】。仍未收到时联系平台支持人员。

### 2.3 找不到项目怎么办？

可能原因：尚未加入项目、项目被删除、当前账号权限不足、当前处于错误的组织或团队层级。

处理方式：联系项目管理员添加成员，检查当前所在组织或团队，请管理员确认项目是否存在并确认账号角色和权限。

### 2.4 无法新建项目怎么办？

当前账号可能不是超级管理员，或没有项目创建权限。请联系超级管理员创建项目，或联系运维人员开通权限。

### 2.5 找不到需要的组件怎么办？

可能原因：当前层级无组件权限，组件尚未从组织层或团队层拉取，项目已存在该组件但未在当前入口显示。

处理方式：在资产广场按组件类型筛选；进入项目【设置】->【资产获取】查看已获取组件；联系管理员从组织层或团队层拉取组件。

### 2.6 上传数据集失败怎么办？

可能原因：文件格式不支持、文件大小超过限制、文件内容为空或损坏、文件命名不符合要求、网络连接异常。

处理方式：检查文件格式，检查数据集名称是否符合 2-36 位字符规则，拆分或压缩过大的文件，重新导出文件后上传，联系技术支持排查。

### 2.7 标签标注任务打不开怎么办？

可能原因：当前账号不是任务负责人、任务已终止、图片资源或标签配置异常、浏览器兼容性问题。

处理方式：检查待办列表是否有该任务，联系任务创建人确认负责人配置，检查任务状态，使用 Edge 或 64 位 Chrome 重新打开。

### 2.8 工作流无法运行怎么办？

可能原因：输入输出资源未配置完整、组件参数未保存、项目内无可用设备、当前账号没有运行权限。

处理方式：检查工作流节点连接是否完整，检查所有输入输出资源是否已配置，点击组件确认参数配置并保存，确认项目内有可用设备，联系管理员检查权限。

### 2.9 本地上传模型无法用于预测怎么办？

AI-PLAT 暂不支持对本地上传模型进行预测操作，仅支持对 AI-PLAT 生产出的模型进行预测。请先使用 AI-PLAT 完成模型训练，再使用训练产出的模型创建预测工作流。

### 2.10 快速 Demo 无法测试怎么办？

可能原因：模型未发布、模型未上架、快速部署服务未启动成功、快速 Demo 类型不支持当前模型、单次上传图片数量超过限制。

处理方式：确认模型已发布并处于上架状态，检查快速部署详情页运行状态，目标检测模型优先使用快速 Demo，单次上传图片不超过 10 张，查看运行日志或联系运维人员。
